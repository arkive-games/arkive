# Map-name i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make map titles in the selector localize (en / zh-CN / zh-TW) by sourcing them from the game L10N export, and fix map visibility (show World_L_B / World_D_B, hide the deprecated Abyss_Reshanta_B).

**Architecture:** A new pure resolver `map_title()` in `extract.py` reads each map's `Desc.Key` from `Map.json` (with a `STR_Map_<name>` fallback for maps absent from `Map.json`) and resolves it through the existing `L10N` loader. The emitter (`emit_frontend.py`) composes the display name as `L10N base + faction suffix` (derived from the machine name) and writes the three locale files. No frontend code changes — `maps` is already a game-data namespace loaded from `data/`.

**Tech Stack:** Python 3.11 (uv project at `tools/`, venv `tools/.venv`), `opencc-python-reimplemented` (zh-TW⇄zh-CN), pytest 8 (dev dep). Frontend: react-i18next (unchanged).

```python
def map_title(name: str, l10n: L10N) -> dict[str, str]:
    """Localized map title from ``Map.json``'s ``Desc.Key``.

    Maps absent from ``Map.json`` (e.g. ``World_L_B`` / ``World_D_B``) fall back
    to the conventional ``STR_Map_<name>`` L10N key. Returns ``{"en", "zhCN"}``;
    values may be empty strings when the key has no L10N body.
    """
    entry = _maps_index().get(name) or {}
    desc_key = (entry.get("Desc") or {}).get("Key") or f"STR_Map_{name}"
    return {"en": l10n.en(desc_key), "zhCN": l10n.zh_cn(desc_key)}
```

(`L10N` is already imported at the top of `extract.py`: `from .l10n import L10N`.)

- [ ] **Step 3b: Add the faction + display helpers**

In the same file, add the following just **above** the `build_maps_yaml` function (it currently starts at line 241, `# --- Builders ---`). Place these new definitions before that comment:

```python
# Faction suffix appended to the L10N base title, keyed by faction. Derived from
# the machine name; Abyss/Reshanta maps (no _L_/_D_) get no suffix. zh-TW is
# produced later by the existing _to_tw() pass over the zh-CN string.
_FACTION_SUFFIX = {
    "light": {"en": " (Elyos)", "zh-CN": "（天）"},
    "dark": {"en": " (Asmodian)", "zh-CN": "（魔）"},
}


def _faction(name: str) -> str | None:
    """Faction from the machine name: ``_L_`` -> light, ``_D_`` -> dark, else None."""
    if "_L_" in name:
        return "light"
    if "_D_" in name:
        return "dark"
    return None


def _map_display(name: str, l10n: L10N) -> dict[str, str]:
    """Composed map display names: L10N base title + faction suffix.

    Returns ``{"name_en", "name_zhCN"}`` (the shape ``_locale_block`` expects).
    Base falls back to ``name.replace("_", " ")`` when L10N has no body; zh base
    falls back to the en base when there is no zh body.
    """
    title = map_title(name, l10n)
    base_en = title["en"] or name.replace("_", " ")
    base_zh = title["zhCN"] or base_en
    fac = _faction(name)
    if fac:
        base_en += _FACTION_SUFFIX[fac]["en"]
        base_zh += _FACTION_SUFFIX[fac]["zh-CN"]
    return {"name_en": base_en, "name_zhCN": base_zh}
```
